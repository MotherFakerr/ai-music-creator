# 里程碑 4: AI 续写旋律

## 目标

基于当前 Channel 已有的音符，AI 自动续写后续旋律。

## 功能描述

### 1. 触发方式

- 在 PianoRoll 界面添加 "AI 续写" 按钮
- 点击按钮后，弹出输入框让用户输入提示词（可选）
- 默认续写到当前 channel 结尾

### 2. 工作流程

```
用户点击 "AI 续写" → 输入提示词(可选) → 发送 MIDI 数据给 AI → AI 返回续写音符 → 合并到 sequencer
```

### 3. AI 接入

**模型选择**：Minimax o3-mini（便宜、支持 JSON 输出）

**Prompt 模板**：
```
你是一个音乐生成助手。基于下面的 MIDI 音符数据，续写 4-8 小节的旋律。

现有音符（MIDI 格式，每行一个音符：pitch, start_step, length, velocity）：
{pitch}, {start}, {length}, {velocity}
...

提示词：{user_prompt}

请返回续写的音符数据，格式为 JSON 数组：
[{"pitch": 60, "start": 16, "length": 2, "velocity": 100}, ...]
```

**响应处理**：解析 JSON → 转换为 PianoRollNote → 追加到当前 channel

### 4. UI 交互

- 按钮位置：PianoRoll 工具栏
- 输入方式：Mantine TextInput 弹窗
- 加载状态：按钮显示 loading，生成完成后自动刷新 PianoRoll

### 5. 参数

| 参数 | 默认值 | 说明 |
|-----|-------|------|
| 续写长度 | 8 小节 | 可配置 |
| 风格提示 | 用户输入 | 可选 |

## 不做什么

- ❌ 实时流式生成（先做批量）
- ❌ 多 channel 同时续写
- ❌ 完整歌曲生成（仅续写）
