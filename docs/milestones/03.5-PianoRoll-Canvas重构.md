# 里程碑 3.5: PianoRoll Canvas 重构 ✅ 已完成

**完成日期**: 2026-02-23

## 目标

将 PianoRoll 从纯 DOM 渲染改为 Canvas + DOM 混合渲染，解决大量音符时的性能问题，同时保持现有交互体验。

## 渲染分层

| 层级 | 渲染方式 | 说明 |
|-----|---------|------|
| Pitch Label | DOM | 最左边显示音阶名，固定不随横向 scroll 移动 |
| 网格线 | Canvas | 横向格子 + 黑键/白键背景区分 |
| Bar Guide | Canvas | 小节分隔线 |
| 音符 | Canvas | 拖拽/缩放/选中态 |
| 框选框 | Canvas | 选区高亮 |
| 播放头 | Canvas | 实时移动 |
| Scrollbar | 原生 | 保持不变 |

## 技术方案

### 1. 容器结构

```
┌────────────┬────────────────────────────┐
│ Pitch Label│      Canvas (绘图层)       │ ← 随 scroll 移动
│   (DOM)    │                            │
│   (固定)   │                            │
└────────────┴────────────────────────────┘
```

### 2. Canvas 尺寸计算

- 使用 `devicePixelRatio` 处理高清屏
- Canvas 尺寸 = 视口尺寸（不是内容尺寸）
- 通过 `scrollLeft/scrollTop` 计算绘制偏移

### 3. 事件处理

- 鼠标事件监听在 DOM 容器上
- 计算坐标：`step = (clientX - rect.left + scrollLeft) / stepWidth`
- 通过 `requestAnimationFrame` 进行重绘

### 4. 性能优化

- 只绘制可见区域（viewport culling）
- 拖拽/缩放时实时预览
- 使用 `ref` 缓存选中状态，避免 React 闭包陷阱
